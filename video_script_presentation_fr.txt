LE PROBLEME ET LA SOLUTION
Le probleme est simple.
Quand on gere des brevets sur papier ou sur Excel, on perd du temps.
On peut faire des erreurs.
Et c'est difficile de retrouver une information.

Ma solution est une application web.
Elle centralise toutes les donnees.
On peut ajouter, modifier, supprimer, et consulter les informations.
On peut aussi voir des statistiques et telecharger des rapports en PDF.


POUR QUI (UTILISATEURS CIBLES)
Cette application peut servir a :
- Un service qui suit des brevets (par exemple une entreprise ou un cabinet).
- Une equipe de recherche.
- Une universite, pour un projet de gestion et de suivi.


IMPACT DANS LA VIE REELLE
Avec cette application :
- On gagne du temps.
- Les donnees sont mieux organisees.
- On peut faire des bilans avec des graphiques.
- On peut generer des rapports PDF rapidement.


LE MODELE DE DONNEES (LES INFORMATIONS GEREES)
Dans mon projet, je gere 5 grandes informations.

1) Domaine
C'est une categorie.
Exemple : informatique, medecine, energie, etc.

2) Invention
Une invention, c'est l'idee ou la creation.
Elle a une description, un resume, et un domaine.

3) Entreprise
Une entreprise a un nom, une activite, un chiffre d'affaires, et une ville.

4) Inventeur
Un inventeur est une personne.
Il a un nom, un prenom, un email, une adresse, et une date de naissance.
Il est rattache a une entreprise.

5) Brevet
Un brevet, c'est l'enregistrement officiel.
Il a une description et deux dates.
Il est lie a une invention et a un inventeur.

TECHNOLOGIES UTILISEES (EXPLIQUEES SIMPLEMENT)
Java 11
Le langage principal pour le code.

JSP (Java Server Pages)
Ce sont les pages web.
Elles affichent les donnees.

Servlets
Ce sont des classes Java qui recoivent les demandes du navigateur.
Elles decident quelle page afficher.
Elles font aussi les actions : ajouter, modifier, supprimer.

JSTL
Une bibliotheque pour faire des boucles et des conditions dans les JSP.
Par exemple : afficher une liste.

Hibernate (JPA)
Un outil qui aide Java a parler avec la base de donnees.
On travaille avec des objets Java.

MySQL (base de donnees SQL)
SQL veut dire : base de donnees en tables (lignes et colonnes).
Ici, la base s'appelle MySQL.

Maven
Un outil pour telecharger les bibliotheques et compiler le projet.

Tomcat
Le serveur web Java qui fait tourner l'application.

Bootstrap + CSS
Pour le design et la mise en page.

JavaScript + jQuery
Pour des interactions simples cote navigateur.

Chart.js
Pour dessiner les graphiques du tableau de bord.

Gson
Pour envoyer des donnees en JSON.
JSON est un format texte pour echanger des donnees.

JasperReports
Pour generer des rapports en PDF.

Lombok
Pour reduire le code repetitif dans les classes.

STRUCTURE DU PROJET (FRONTEND / BACKEND)
Le code est separe en deux grandes parties.

PARTIE FRONTEND (src/main/webapp)
Elle contient tout ce que le navigateur voit :
- index.jsp : page d'entree. Elle redirige vers /home.
- WEB-INF/views : toutes les pages JSP (liste, ajout, modification).
- WEB-INF/jspf : morceaux reutilisables (header et footer).
- css, js, images, bootstrap : fichiers statiques (style, scripts, images).
- WEB-INF/Reports : fichiers de rapports pour les PDF.

PARTIE BACKEND (src/main/java)
Elle contient la logique :
- controllers : les servlets (controleurs).
- models : les classes (Brevet, Invention, Inventeur, Entreprise, Domaine).
- dao : acces base de donnees (operations : ajouter, lire, modifier, supprimer).
- metier : couche service entre controllers et dao.
- util : classes utilitaires (constantes, rapports, donnees pour les graphiques).
- hibernate.cfg.xml : configuration de connexion a MySQL.


COMMENT LE FRONTEND COMMUNIQUE AVEC LE BACKEND
1) Navigation
Quand je clique sur le menu, le navigateur envoie une demande au serveur.
Exemple : /brevets?mode=list.
Le controleur lit le parametre "mode".
Il choisit la bonne page JSP.

2) Formulaires
Les formulaires envoient une demande POST.
Ils ont un champ cache "op" (add ou update).
Le controleur lit les champs, puis enregistre dans la base.

3) Donnees pour les graphiques
Sur le tableau de bord, JavaScript envoie une demande a /ChartController.
Le serveur renvoie du JSON.
Ensuite Chart.js construit les graphiques.

4) Rapports PDF
Le bouton "Imprimer" appelle /ReprotsController?filename=...
Le serveur genere un PDF et le telecharge.
